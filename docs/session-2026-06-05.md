# IAT Forms Portal — Build Session · June 5, 2026

## What We Built

### 1. Forms Portal — Bento Card UI
Replaced the plain text list inside category accordions with a product-style bento grid layout.

- **Three card sizes**: featured (2×2), wide (2×1), default (1×1) — first card in each category is always featured, every 5th is wide, rest are default
- **Category-colored accents**: each card gets hover border color, icon tint, and a subtle radial glow matching its category (blue, amber, violet, sky, emerald)
- **Watermark icon**: large ghost icon in the corner of featured and wide cards
- **Sort dropdown** per accordion and on individual category tabs: Most Used (default), A → Z, Z → A — sort order is backed by real submission counts fetched from the database
- Search results and single-category tab views also use the bento grid

---

### 2. Employee Portal — Full Build

A brand new employee-facing section at `/employee/*`, completely separate from the admin portal.

#### Auth
- Supabase Auth (email + password) via `@supabase/ssr`
- Middleware protects all `/employee/*` routes except `/employee/login` and `/employee/welcome`
- Admin portal keeps its existing cookie-based auth — untouched

#### Account Creation (Admin Side)
- `/admin/employees` → **Add Employee** modal
- Admin enters name, email, title, department, admin flag
- A strong temporary password is auto-generated (admin can regenerate or type their own)
- Account created directly via Supabase Admin API (`createUser` with `email_confirm: true`) — **no email sent**, no rate limits
- Success screen shows credentials with one-click copy; warns password won't be shown again

#### First-Login Onboarding (`/employee/welcome`)
- Triggered automatically on first login (`setup_complete` flag not set in auth metadata)
- **Step 1 — Set Password**: email pre-filled and locked, live password requirements checker (8+ chars, uppercase, lowercase, number, special character — each turns green as fulfilled, confirm field border turns red/green in real time)
- **Step 2 — Complete Profile**: name, job title, department, phone, bio — all optional except name
- On submit: password set via `supabase.auth.updateUser`, profile saved, `setup_complete` flagged — future logins go straight to profile

#### Employee Profile (`/employee/profile`)
- Editable: name, job title, department, phone, bio/hobbies
- Read-only (admin-controlled): email, PTO balance, sick time balance
- Balance display cards show current hours + accrual rate per pay period

#### Time Off Requests (`/employee/requests`)
- Submit PTO or sick time: type toggle, hours, date range, optional notes
- Live overdraw warning if requested hours exceed available balance
- Request history with status badges (Pending / Approved / Denied)
- Balance summary cards at top

#### Employee Directory (`/employee/directory`)
- Bento grid of all employees — same featured/wide/default sizing
- Each employee gets a deterministic color from an 8-color palette based on their ID (consistent across sessions)
- **Profile modal**: click any card → slide-up modal with color strip header, avatar rings out of the strip, info rows (email, phone, department, title, hire date), and an "About" section with bio/hobbies

### 3. Admin Portal — Extensions

#### Employees (`/admin/employees`)
- Lists all employees with PTO/sick balances, admin badge
- Invite → **Add Employee** modal (described above)
- Click through to employee detail

#### Employee Detail (`/admin/employees/[id]`)
- Edit all fields including email (admin-only), balances, accrual rates, hire date, admin flag
- Manual balance adjustments logged to `accrual_log` with reason and delta
- Request history sidebar showing last 20 requests with status

#### Time Off Queue (`/admin/requests`)
- Filterable by status: Pending / Approved / Denied / All
- Each request card shows employee, type, hours requested, current balance, dates, notes
- Overdraw warning (amber highlight) when requested hours exceed balance
- **Approve / Deny** buttons — approval deducts balance, logs to `accrual_log`, emails employee; denial emails employee

### 4. Email Notifications (via Resend)

| Trigger | Recipient | Notes |
|---|---|---|
| Request submitted | All admin employees + `ADMIN_NOTIFICATION_EMAIL` fallback | Overdraw warning in subject + amber header if insufficient balance |
| Request approved | Employee | Balance deduction confirmed |
| Request denied | Employee | Soft denial message |

- Emails fully awaited before response — no Vercel serverless early-termination
- `ADMIN_NOTIFICATION_EMAIL` env var as guaranteed fallback regardless of `is_admin` flags

---

### 5. Database Schema (Supabase)

Three new tables added via migration `001_employee_pto_system.sql`:

**`employees`** — linked to `auth.users` via trigger on signup
- `pto_balance`, `sick_balance` (numeric, admin-controlled)
- `pto_accrual_rate`, `sick_accrual_rate` (default 4 hrs / 2 hrs per pay period)
- `is_admin`, `hire_date`, `bio`, `phone`, `department`, `job_title`, `avatar_url`

**`time_off_requests`**
- `type` (pto | sick), `hours_requested`, `start_date`, `end_date`, `status` (pending | approved | denied)
- `reviewed_by`, `reviewed_at`

**`accrual_log`** — audit trail for every balance change
- `hours_delta` (positive = added, negative = deducted), `reason`, `note`

RLS policies on all three tables. Trigger auto-creates `employees` row on `auth.users` insert.

### 6. API Routes Added

| Method | Route | Auth | Purpose |
|---|---|---|---|
| `GET` | `/api/employees/me` | Employee session | Fetch + auto-create own employee record |
| `PATCH` | `/api/employees/me` | Employee session | Update own editable profile fields |
| `GET` | `/api/employees` | Employee session | List all employees (directory) |
| `POST` | `/api/employees/invite` | Admin cookie | Create employee account (no email) |
| `PATCH` | `/api/employees/[id]` | Admin cookie | Admin update any employee field |
| `GET` | `/api/requests` | Employee session | Fetch own requests |
| `POST` | `/api/requests` | Employee session | Submit new request + notify admins |
| `POST` | `/api/requests/[id]/review` | Admin cookie | Approve or deny, deduct balance, email employee |
